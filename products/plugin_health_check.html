Frequently Asked Questions

Does this plugin replace the built-in WordPress update checker?

No, it complements it. WordPress shows updates when they are available. Plugin Health Check identifies plugins that might not have had updates released recently, potentially indicating they are unmaintained or abandoned.

How often does the plugin check for stale plugins?

It performs a check periodically using WordPress cron jobs (typically once daily). Administrators can also trigger a manual check from the settings page.

Can I ignore specific plugins from the checks?

Yes, the settings page includes an option to select and exclude specific plugins from the health check scan.

Is the automatic deactivation feature safe?

The automatic deactivation is optional and highly configurable. It's designed as a safety net after a significant grace period. It's recommended to review the list of stale plugins manually before relying on automatic deactivation, especially on critical sites.

Does this plugin send data to external servers?

No, it only fetches public information from the official WordPress.org API to compare plugin versions. No sensitive site data is transmitted. Email notifications are sent from your own server if enabled.
